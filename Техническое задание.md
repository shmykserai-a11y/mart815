# Техническое задание: Сайт-квест «Призрак в консоли»

## Общая концепция
Одностраничный сайт, который имитирует хакерский терминал с элементами ARG. Сайт имеет два состояния:
- **заблокированное** (фейковая страница ошибки браузера)
- **активированное** (полноценный терминал)

Переход между состояниями осуществляется через консоль разработчика в браузере.

---

## Состояние 1: Фейковая страница ошибки (Locked State)

### Внешний вид
- Полная имитация стандартной страницы ошибки Google Chrome «Не удается получить доступ к сайту».


### Секретные особенности (отличия от настоящей страницы ошибки)
1. **Курсор в углу**: в левом нижнем углу страницы едва заметно мигает зелёный курсор (символ подчёркивания `_`) с цветом `#00ff00` и прозрачностью `0.3`. Мигание раз в секунду.
2. **Скрытый текст при выделении**: если выделить весь текст на странице (`Ctrl+A`), в самом низу, за пределами видимой области, появляется скрытый блок с текстом:
   ```html
   <!-- Система заблокирована. Требуется ручное вмешательство. Проверьте консоль. -->
   ```
3. **Консоль браузера**: при открытии консоли разработчика (`F12`) в ней выводится кастомное сообщение красным цветом:
   ```
   Uncaught Error: System.Locked() - User verification failed.
   ```
   И сразу под ним, серым цветом:
   ```
   > Чтобы разблокировать, введи: window.userTrust = true
   ```

### Поведение
- Страница не перезагружается автоматически.
- Кнопка «Обновить» работает, но возвращает в это же состояние.
- Любые попытки взаимодействия с элементами страницы (кроме консоли) бесполезны.

### Триггер разблокировки
Как только пользователь вводит в консоли команду:
```javascript
window.userTrust = true
```
страница плавно (с эффектом затухания/перехода) меняет содержимое на **Состояние 2**.

## Состояние 2: Терминал (Unlocked State)

### Общая стилистика
- Чёрный фон (`#000000` или `#0a0a0a`).
- Зелёный текст (`#00ff00` или `#33ff33`).
- Моноширинный шрифт: `'Courier New', Courier, monospace`.
- Эффект «старого монитора»: лёгкое свечение текста, минимальное мерцание.
- При загрузке терминала происходит эффект печатающегося текста (символ за символом, задержка 30–50 мс).

### Интерфейс терминала

**Шапка терминала:**
```
==================================================
   SYSTEM TERMINAL v2.4.1 // ДЕНЬ РОЖДЕНИЯ ПРОТОКОЛ
==================================================
```

**Приветственное сообщение (печатается с эффектом):**
```
> Добро пожаловать, гость.
> Система разблокирована. Уровень доступа: Гость (Read-Only)
> Доступные команды: help, play, scan, admin
> 
> Введите 'help' для списка команд или 'play' для прослушивания сообщения.
```

**Поле ввода команд:**
- Строка ввода начинается с приглашения: `guest@birthday:~$ `
- Курсор мигает после приглашения.
- Команды вводятся с клавиатуры, выполняются по нажатию `Enter`.
- История команд не сохраняется (можно добавить опционально).

### Команда: `help`

**Вывод:**
```
> Доступные команды:
>   help            - показать эту справку
>   play            - воспроизвести аудиосообщение
>   scan            - открыть анализатор сигнала
>   admin           - вход в режим администратора
>   
> Для перехода к анализу спектра используй 'scan'.
```

### Команда: `play`

**Функционал:**
- Воспроизводит аудиофайл, встроенный в сайт (например, в папке `/assets/` или в base64).
- Аудиофайл: 10–15 секунд, голос, записанный с эффектом «робота» или замедленный.
- При воспроизведении в интерфейсе появляется индикатор:
  ```
  > Воспроизведение: [████████░░░░] 45%
  > Длительность: 00:15
  ```
- После окончания воспроизведения сообщение:
  ```
  > Сообщение доставлено. Рекомендуется провести спектральный анализ.
  > Используй 'scan' для открытия анализатора.
  ```

### Команда: `scan` — ключевой функционал

Терминал очищается и показывает интерфейс анализатора:

**Важное требование к спектрограмме:**
Это не настоящий аудиоанализ в реальном времени, а **заранее подготовленное изображение ASCII-графикой**, которое визуально имитирует спектрограмму. При этом на определённой «частоте» (например, на уровне 2100 Гц, соответствующем 7-й строке сверху) должно быть визуально различимо число — пароль от админки. Число составляется из символов `█` и `▓` так, чтобы его можно было прочитать глазами (например, `2 2 0 3 1 9 9 5`).

**Интерактивность:**
- Нажатие `r` перезапускает аудио (через скрытый элемент `<audio>`).
- Нажатие `v` переключает между 2–3 вариантами ASCII-графики для красоты (пароль должен оставаться читаемым хотя бы в одном из режимов).
- Нажатие `Esc` возвращает в обычный режим терминала с сохранением состояния.

### Команда: `admin`

**Поведение:**
При вводе `admin` система запрашивает пароль:
```
> Вход в режим администратора.
> Введите пароль: [_____]
```
Символы пароля скрываются звёздочками или точками.

**Проверка пароля:**
- Если пароль неверный:
  ```
  > Неверный пароль. Попытка залогирована.
  > Осталось попыток: 2
  ```
- Если пароль верный (тот, что был в спектрограмме):
  ```
  > ДОСТУП РАЗРЕШЕН.
  > Здравствуй, Администратор.
  > Загружается протокол "Подарок"...
  > [██████████████████████████░░░░] 80%
  > 
  > Ошибка! Файл подарка был перемещен.
  > Последняя известная геолокация:
  > Широта: 55.751244
  > Долгота: 37.618423
  > (Координаты места, где спрятан подарок)
  >
  > Для подтверждения получения введи 'accept'
  ```

### Команда: `accept`

После ввода `accept`:
```
> Статус: Подтверждено.
> Подарок активирован. Система завершает работу.
> С днём рождения, [Имя]!
> 
> Сеанс будет закрыт через 10 секунд...
```
Через 10 секунд сайт либо показывает большое поздравительное сообщение, либо перенаправляет на видео‑поздравление, либо просто выводит мигающую надпись `GIFT ACTIVATED`.

## Дополнительные элементы и атмосфера

### Случайные «шумы» в терминале
- Раз в 30–60 секунд появляется строка системного лога:
  ```
  [SYS] Проверка целостности: OK
  ```
  или
  ```
  [NET] Пинг 8.8.8.8: 42ms
  ```

### Эффект печатающегося текста
- Все сообщения при первой загрузке терминала должны печататься с задержкой (30–50 мс на символ).

### Адаптивность
- Сайт должен корректно отображаться на мобильных устройствах (основной упор на десктоп, но базовая вёрстка должна быть резиновой).
- На мобильных устройствах консоль разработчика недоступна, поэтому стоит добавить альтернативный способ: скрытая кнопка в углу, которая появляется после 5 быстрых тапов по экрану.

### Цветовая схема
- Фон: `#000000`
- Текст: `#00ff00`
- Акценты (рамки, заголовки): `#33ff33` или `#66ff66`
- Ошибки: `#ff5555` (красный)
- Системные сообщения: `#888888` (серый)

### Аудиофайл
- Встроен в сайт (например, в папке `/audio/`).
- Формат: MP3 или WAV.
- Содержание: твой голос, говорящий примерно так: *«Привет! Если ты это слышишь, значит ты прошла первый уровень. Но настоящий ключ спрятан там, где звук становится картинкой. Ищи в спектре.»*
- Голос можно обработать эффектом «робота», замедлить или инвертировать так, чтобы разобрать слова без спектрограммы было трудно.

## Технические требования для реализации (без указания стека)

1. **HTML-структура**:
   - Один корневой элемент `<div id="app">`.
   - Два состояния переключаются через CSS-классы или JavaScript.
   - Все элементы генерируются динамически.

2. **CSS**:
   - Анимация мигания курсора.
   - Эффект свечения текста.
   - Адаптивная вёрстка.

3. **JavaScript**:
   - Обработка ввода команд (слушатель клавиатуры).
   - Эмуляция печатающегося текста.
   - Переключение между состояниями (терминал / анализатор).
   - Проверка пароля.
   - Воспроизведение аудио.
   - Обработка команд `help`, `play`, `scan`, `admin`, `accept`.
   - Обнаружение ввода `window.userTrust = true` в консоли.

4. **Аудио**:
   - Элемент `<audio>` с предзагруженным файлом.
   - Контроль воспроизведения через JavaScript.

5. **Спектрограмма**:
   - Статичный ASCII-арт, который меняется при нажатии `v`.
   - В одном из режимов явно читается пароль.

## Пользовательский путь (итоговый сценарий)

1. Пользователь открывает сайт → видит фейковую ошибку браузера.
2. Интуитивно открывает консоль (`F12`).
3. Видит подсказку, вводит `window.userTrust = true`.
4. Сайт трансформируется в терминал.
5. Пользователь вводит `help`, затем `play` (слушает аудио).
6. Вводит `scan`, видит ASCII-спектрограмму, находит в ней пароль.
7. Возвращается в терминал (`Esc`), вводит `admin` и пароль.
8. Получает координаты подарка.
9. Вводит `accept` и видит финальное поздравление.
